CSS Позиционирование

Cтатичное позиционирование

position:static

По умолчанию позиционирование всех элементов на веб-странице статичное position:static, это означает, что элементы находятся в обычном потоке документа html, то есть идут друг за другом, согласно написанию в коде.

В реальной жизни, вы не будете использовать это значение пока вам не потребуется переопределить ранее заданное позиционирование на статичное:

#div-1 {
  position: static;
}

Относительное позиционирование

position:relative

Применяется если вам необходимо какой-то элемент веб-страницы переместить относительно его начального расположения в документе на другое место.

Свойствами top (верх) и bottom (низ) вы задаете смещение по вертикали от нормального положения элемента, свойствами left (лево) и right (право) — вы  можете установить горизонтальное смещение.

Давайте переместим блок div-1 на 20 пикселей вниз и на 40 пикселей влево:

#div-1 {
 position: relative;
 top: 20px;
 left: -40px;
}

Посмотрите на пространство, которое ранее занимал блок div-1, перед тем, как мы его переместили: теперь там пустое пространство. Оно не схлопнулось. Следующий в документе элемент div-after никуда не сдвинулся, когда мы сместили div-1. Это произошло потому, что пространство по-прежнему в документе занимает div-1, несмотря на его перемещение и относительно этого места отсчитываются коодинаты смещения.

Cвойство position:relative выглядит не слишком полезным в данном случае. Но оно еще пригодится нам для других задач.

Абсолютное позиционирование

position:absolute

Когда вы задаете значение position:absolute, элемент извлекается из стандартного потока документа и размещается там, где вы ему укажете.

Давайте переместим блок div-1a в правый верхний угол страницы:

#div-1a {
 position: absolute;
 top: 0;
 right: 0;
 width: 200px;
}

В этот раз, когда мы убрали div-1a из потока документа, другие элементы расположились на странице по другому: div-1b, div-1c и div-after переместились в потоке выше, так как место, занимаемое div-1a освободилось.

Вы можете заметить, что div-1a расположился в правом верхнем углу веб-страницы. Хорошо, что мы можем располагать вещи относительно всего документа, но это имеет ограниченное применение в реальной жизни.

Что мы хотим на самом деле, так это расположить блок div-1a относительно div-1. И в этом нам поможет предыдущее свойство position:relative.

Как работает абсолютное позиционирование:

Элемент удаляется из обычного потока документа, и на месте элемента в макете не остается пустого места.

  • Он будет расположен относительно его ближайшего позиционированного предка*, если таковой имеется;
  • в противном случае он помещается относительно начального содержащего блока (области <html>, представляющей вьюпорт страницы).

Если свойство position имеет значение absolute, то содержащий блок задаётся краем padding бокса *ближайшего позиционированного предка (у которого, свойство position имеет значение fixed, absolute, relative или sticky).

Cодержащий блок может также задаваться краем padding бокса ближайшего предка, у которого:

  • Свойство transform или perspective имеет значение отличное от none.
  • Свойство will-change имеет значение transform или perspective.
  • Свойство filter имеет значение отличное от none или will-change (работает только в Firefox).
  • Свойство contain имеет значение paint (например, contain: paint;).

Его конечная позиция определяется значениями top, right, bottom, и left.

Относительное и абсолютное позиционирование

position:relative
+
position: absolute

Абсолютное позиционирование:

  • Элемент будет расположен относительно его ближайшего относительно позиционированного предка (у которого, свойство position имеет значение fixed, absolute, relative или sticky), если таковой имеется;

Если мы зададим position:relative для div-1, то любые абсолютно спозиционированные элементы внутри div-1 будут позиционироваться относиельно div-1.

Затем, если мы зададим абсолютное позиционирование position:absolute для div-1a, мы сможем переместить его в верхний правый угол div-1:

#div-1 {
 position:relative;
}
#div-1a {
 position:absolute;
 top:0;
 right:0;
 width:200px;
}

Таким образом, если вы хотите контролировать позиционирование одного блока относительного другого, то вы можете использовать этот прием родитель — relative, дочерние элементы — absolute.

Вам совершенно необязательно куда-то перемещать родителя относительно его начальных координат. Если вы не задаете координат, relative — родительский блок остается на своем месте в потоке, а его дочерние absolute — элементы вы можете располагать как вам захочется уже относительно него.

Кроме того, вы также можете родителя — делать не только relative, если это будет необходимо (fixed, absolute, relative или sticky).

То, есть возможно позиционирование relative > absolute > absolute.
Каждый дочерний absolute - элемент, располагается относительно своего ближайшего спозиционированного родителя.

Фиксированное позиционирование

position:fixed

Элемент выбивается из обычного потока документа, и  не занимает пространство среди обычных элементов документа.

В данном примере не имеет значения тип позиционирования div-1.

Он позиционируется относительно viewport исходного содержащего блока (<html>), за исключением случаев, когда один из его предков имеет свойство transform, perspective, или filter, установленное на что-то иное, кроме none (см. CSS Transforms Spec). В этом случае этот предок поведёт себя как содержащий блок.

Проще:
Если вы сделаете элемент fixed, а к его родителю примените фильтр или трансформацию - то фиксированное позиционирование переключится с вьюпорта всего документа на размер родителя с трансформацией или фильтром!

#div-1 {
 position:relative;
}
#div-1a {
 position:fixed;
 top: 50%;
 left: 50%;
 width: 200px;
}

Позиция фиксированно спозиционированного элемента определяется значениями top, right, bottom, и left.

В печатных документах элемент помещается в одно и то же положение на каждой странице.

Фиксированное позиционирование + трансформация

position:fixed
+
transform: translate(-50%,-50%);

Если вы хотите фиксированно спозиционировать элемент по центру веб-страницы относительно вьюпорта, вы можете для фиксированного элемента задать координаты 50% от верхней и левой границ. При этом, позиционирование будет вестись от левого верхнего угла элемента.

Для компенсации координат центра вы можете применить трансформацию со смещением элемента на половину ширины влево и на половину высоты наверх. Так как координатные оси отсчета идут сверху вниз и слева направо, то процентные значения будут отрицательными.

#div-1a {
 position:fixed;
 top: 50%;
 left: 50%;
 width: 200px;
 transform: translate(-50%,-50%);
}

«Прилипающее» позиционирование

position:sticky

При задании позиционирования sticky элемент залипает, когда область просмотра достигает определённой позиции.

«Липкое» позиционирование состоит из двух основных частей: «липкого» элемента и «липкого» контейнера.

  • «Липкий» элемент — это элемент, которому мы задали position:sticky. Элемент будет становиться плавающим, как только область видимости достигнет определённой позиции, например top:0.
  • «Липкий» контейнер — это HTML-элемент, который оборачивает «липкий» элемент. Это максимальная область, в которой может перемещаться наш элемент.

Когда вы задаёте элементу position: sticky, его родитель автоматически становится «липким» контейнером!

Контейнер будет являться областью видимости для элемента. «Липкий» элемент не может выйти за пределы своего «липкого» контейнера.

«Липкий» элемент не будет залипать если он — единственный дочерний элемент контейнера.

#sticky{
  position: sticky;
  top: 0;
}

position: sticky ведёт себя не так, как другие типы позиционирования. Но, с другой стороны, у них есть определённые сходства:

  • relative или static — «липкий» элемент похож на элемент со статическим или относительным позиционированием поскольку сохраняет свою естественную позицию в DOM (остаётся в потоке).
  • fixed—когда элемент залипает, то ведёт себя как будто у него заданы стили position: fixed, остаётся на той же позиции в области видимости и вырывается из потока документа.
  • absolute—в конце доступной для перемещений области элемент останавливается и остаётся поверх другого элемента. Точно также, как ведёт себя абсолютно спозиционированный элемент в контейнере с position: relative.

Позиционирование элементов по оси z

z-index

Свойство z-index определяет положение элемента и нижестоящих элементов по оси z.

В случае перекрытия элементов, это значение определяет порядок наложения. В общем случае, элементы с большим z-index перекрывают элементы с меньшим.

В одном контексте наложения элементы перекрывают друг друга согласно порядку следования в документе, то есть последний элемент — поверх предыдущего.

#div-1 {
   position:relative;
  }
  #div-1a {
   position:absolute;
   top:0;
   left:10px;
   width:200px;
   z-index: 1;
  }
  #div-1b {
   position:absolute;
   top:0;
   left:0;
   width:200px;
  }

Используя z-индекс для позиционируемых элементов, мы можем изменить порядок наложения элементов, заданный по умолчанию.

При применении определенных CSS-свойств элемент может формировать контекст наложения.

Контекст наложения (stacking context) — это концепция трёхмерного расположения HTML-элементов вдоль оси Z по отношению к пользователю, находящемуся перед экраном. HTML-элементы занимают это место по порядку, основанному на атрибутах элемента.

	     |	   |	     |	  |
	     |		|    |	  |	⇦ ☻
	     |		|	  |	user
z-index:  canvas  -1	0    1	  2

Значения z-индекса имеют смысл только в одинаковом контексте наложения.
